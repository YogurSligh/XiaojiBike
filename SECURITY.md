# 安全政策

## 支持范围

当前项目处于个人开源阶段，维护者只承诺尽力处理默认分支中的安全问题。自行部署、二次开发、改造为商业服务或接入第三方数据源后的安全责任由部署者承担。

## 报告安全问题

请不要在公开 issue 中披露可被直接利用的漏洞细节。可以先通过私下渠道联系仓库维护者，至少包含：

- 受影响的版本或提交。
- 复现步骤和影响范围。
- 是否涉及密钥、日志、个人信息、服务端请求伪造、任意文件读写、鉴权绕过或供应链风险。

## 部署建议

- 不要把 `.env`、运行缓存、日志、数据库、密钥或第三方数据授权文件提交到仓库。
- 公网部署前应加反向代理限流、访问日志保护、错误信息降敏和健康检查降敏。
- 不要向匿名公网用户开放无限制刷新、批量抓取或高频请求能力。
- 使用正式授权的数据源，或确认第三方服务协议允许对应场景。
