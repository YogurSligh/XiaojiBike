# 开源合规说明

本说明用于帮助维护者和二次部署者在公开仓库或自托管服务前完成基本检查。它不是法律意见。

## 当前项目定位

- 开源学习项目。
- 个人研究工具。
- 不提供投资建议、基金评价、基金销售、代客理财、交易撮合或收益承诺。
- 不内置用户账户、支付、交易、持仓导入或个人资产管理功能。

## 开源发布检查

- 仓库根目录包含 `LICENSE`。
- 仓库根目录包含 `DISCLAIMER.md`。
- 仓库根目录包含 `PRIVACY.md`。
- 仓库根目录包含 `SECURITY.md`。
- 仓库根目录包含 `THIRD_PARTY_NOTICES.md`。
- `README.md` 明确项目定位、免责声明、数据源限制和自托管责任。
- 不提交 `.env`、运行缓存、日志、数据库、密钥、私有授权文件和个人数据。
- 发布前运行 `git diff --check`、后端测试和前端构建。

## 公网部署额外检查

如果把本项目部署成公开网站，建议至少完成：

- 使用合规云资源、域名和备案流程。
- 明确首页免责声明和隐私说明入口。
- 为公开 API 加限流、缓存、超时、错误降敏和基础监控。
- 关闭或限制匿名 `force_refresh` 能力，避免被用作批量抓取代理。
- 使用授权数据源，或确认第三方服务协议允许公开展示、缓存和再分发。
- 不使用“推荐”“优选”“买入”“收益保证”“稳赚”“评级”“排名”等容易被理解为投顾、荐基或基金评价的表达。
- 不接入基金购买、开户、返佣、广告导流、付费会员、打赏解锁、组合推荐或自动交易。

## 数据源边界

本项目代码可调用第三方公开页面或接口，但开源本项目代码不等于获得第三方数据授权。任何公开服务、商业使用、批量抓取、缓存再分发或嵌入其他产品前，都应重新确认数据源协议。

## 维护者变更守则

- 新增金融指标时，同时补充指标含义、数据来源和计算方式。
- 新增排序、评分、默认筛选或高亮规则时，避免形成“推荐买入”含义。
- 新增第三方依赖时，同步更新 `THIRD_PARTY_NOTICES.md`。
- 新增用户输入、日志、分析 SDK 或错误上报时，同步更新 `PRIVACY.md`。
